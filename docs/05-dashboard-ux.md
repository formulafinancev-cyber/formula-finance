# Dashboard UX Principles — Formula Finance v1.0.0

Design and UX guidelines for all dashboards and reports generated by Formula Finance.
Based on research from Smartsheet, Xenia Team, and industry best practices.

---

## Core Design Principles

### 1. Information Hierarchy
- **Level 1**: Critical KPIs (revenue, profit, cash) — top-left, largest
- **Level 2**: Supporting metrics (costs, trends) — middle section
- **Level 3**: Detail / drill-down — bottom, smaller

### 2. Colour Coding (consistent across all dashboards)

| Colour | Meaning | Usage |
|--------|---------|-------|
| Green | Good / On target / Positive | Revenue up, margins good |
| Red | Alert / Below target / Negative | Revenue down, cost over |
| Yellow/Amber | Warning / Near threshold | Approaching limit |
| Blue | Neutral / Informational | Totals, labels |
| Grey | Inactive / No data | Unavailable blocks |
| Dark bg | Header rows | Section titles |

### 3. Layout Grid for Google Sheets

```
Row 1-2:   Dashboard title + Last updated timestamp
Row 3:     Navigation / Tab summary line  
Row 5-8:   Top KPI cards (4-6 across)
Row 10-20: Primary charts / trend tables
Row 22-35: Secondary metrics
Row 37+:   Detail tables
```

### 4. KPI Card Structure

Each KPI card contains:
```
+---------------------------+
| Metric Name               |
| ▲ VALUE      vs Plan: +X% |
| Trend sparkline           |
| ▲ vs prev period: +Y%    |
+---------------------------+

States:
- ACTIVE: shows value + delta + trend
- NO_DATA: shows grey card with [DATA UNAVAILABLE]
- PARTIAL: shows value only, no trend
```

## Dashboard Types

### CEO Dashboard

**Purpose**: Strategic overview for executive decision-making
**Update frequency**: Daily
**Sections**:
1. Revenue vs Plan (current month, YTD)
2. Net Profit + Margin trend
3. Top-3 Revenue channels
4. Cash position
5. Key alerts (red flags)
6. Forecast (next 30/90 days if data available)

**Colour theme**: Dark header with green/red accents
**Max rows visible without scroll**: 40

### CFO Dashboard

**Purpose**: Financial management and control
**Update frequency**: Daily
**Sections**:
1. Cash flow waterfall (in/out by category)
2. P&L Budget vs Actual
3. Debt & AR/AP aging
4. Balance sheet snapshot
5. Cost structure breakdown
6. Cash forecast

**Colour theme**: Professional blue/dark

### Ops Dashboard

**Purpose**: Day-to-day operational performance
**Update frequency**: Daily (or per shift if iiko connected)
**Sections**:
1. Revenue today / this week / MTD
2. Average check + Covers trend
3. Food cost % + Waste %
4. Staff hours + Labor cost %
5. Top/Bottom menu items
6. Inventory alerts (if data available)

**Colour theme**: Clean, operational white/green

### General Dashboard

**Purpose**: Company-wide KPI summary for all stakeholders
**Update frequency**: Weekly
**Sections**:
1. Company scorecard (traffic light — Red/Amber/Green per KPI)
2. Financial summary
3. Operational summary
4. Trend charts (last 12 periods)

## Formatting Standards

### Numbers

```
Revenue:    1 234 567 ₽ (thousands separator, no decimals)
Percentages: 12.3%
Small amounts: 1 234.56 ₽
Growth:     +12.3% / -5.7%
Large sums: 1.2M ₽ / 12.5K ₽  
```

### Dates

```
Default: DD.MM.YYYY  
Month label: Янв 2025, Фев 2025
Week: Нед.1 2025, Нед.2 2025
```

### Cell Formatting Guidelines

```
- Header cells: Bold, dark background (#1a1a2e), white text
- Section headers: Semi-bold, light background (#e8f4f8)
- KPI values: Large font (14-16pt), bold
- Delta values: Coloured (green/red), 11pt
- Data cells: Regular, 10pt, left/right aligned appropriately  
- NO_DATA cells: Grey background, italic text, 10pt
```

## Email Report Design

### HTML Email Template Structure

```html
[Header: Company name + Logo + Period]
[Hero metrics: 3-4 top KPIs with trend arrows]
[Section: Financial Summary table]
[Section: Operational Summary table]
[Section: Alerts (if any)]
[Footer: Generated by Formula Finance + timestamp]
```

### Email Best Practices
- Max width: 600px (mobile-friendly)
- Inline CSS only (Gmail compatibility)
- No JavaScript in emails
- All values pre-computed before sending
- Attach full report as PDF only if explicitly configured
- Subject line: `[Фев 2025] CEO Отчет | Formula Finance`

---

## Accessibility

- Never rely on colour alone — add arrow symbols (▲/▼) for direction
- Include text labels in all KPI cards, not just numbers
- Font size minimum 10pt in sheets
- All dashboard sheets are read-only for non-admin users

---

## NO_DATA Handling

```
If a section has no data:
  - Show grey placeholder box
  - Display: "[DATA UNAVAILABLE - missing: ReportType1, ReportType2]"
  - Do NOT show zeros, averages, or estimates
  - Do NOT calculate % based on missing components
  - Log missing reports to Debug sheet
```
